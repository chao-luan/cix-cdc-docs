# 全镜像烧录

全镜像烧录用于将系统镜像写入开发平台的启动介质或目标存储设备中，包括 U 盘、SD 卡、NVMe SSD、板载存储等。不同开发平台的镜像格式、启动方式、烧录工具和恢复方式可能存在差异，实际操作前请确认设备型号、系统镜像版本和目标存储介质。

!!! warning "注意"
    镜像烧录会覆盖目标存储设备中的原有数据。请在操作前确认烧录目标，避免误擦除本地硬盘、移动硬盘或其他重要数据。

## 适用范围

本章节用于汇总 CIX P1 相关开发平台的系统镜像烧录入口。

当前涉及平台包括：

- 瑞莎 Orion O6
- 美高铭凡 MS-R1
- 香橙派 OrangePi 6 Plus
- 贝启平台

## 常见烧录方式

不同平台支持的烧录方式可能不同，常见方式包括：

1. **U 盘启动 / U 盘安装**  
   将系统镜像或安装镜像写入 U 盘，通过 BIOS / 启动菜单选择 U 盘启动，用于系统安装、恢复或维护。

2. **SD 卡 / TF 卡烧录**  
   将系统镜像写入 SD 卡或 TF 卡，适用于支持 SD 卡启动的开发板平台。

3. **NVMe SSD 烧录**  
   将系统镜像写入 NVMe SSD，适用于以 NVMe 作为主系统盘的开发平台。

4. **板载存储烧录**  
   将系统镜像写入 eMMC、UFS 或其他板载存储介质，具体方式需参考对应平台官方文档。

5. **系统内写盘 / 恢复模式烧录**  
   部分平台支持在已有系统环境、维护系统或恢复模式下，将镜像写入目标磁盘。

## 基本流程

镜像烧录的一般流程如下：

1. 下载对应平台和型号的系统镜像。
2. 校验镜像文件完整性。
3. 准备目标存储介质，例如 U 盘、SD 卡或 NVMe SSD。
4. 使用官方推荐工具或通用写盘工具写入镜像。
5. 将存储介质接入开发平台。
6. 进入 BIOS / 启动菜单，选择对应启动设备。
7. 启动系统后，检查系统版本、磁盘分区和基础外设状态。

具体烧录步骤请以对应平台官方文档为准。
